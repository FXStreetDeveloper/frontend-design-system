Intro

This guide lists the components of FXStreet's main website.

The project is built with LESS and compiled into one single CSS file for the main site.

Aside from the custom LESS files, we also incorporate vendor components from Bootstrap v3.3.5, LESS mixins from Less Hat and icons from Fontawesome.

In order to get started, you have to know the structure of the project, the LESS files and the compiled CSS files

View the GitHub Project

The Project's structure

The components are divided with the Atomic design aproach.

In order to ensure mantainance and scalability, each group of components are divided into different LESS files to import all into the final "fxs_screen.min.css" or use it separately in order to build third partie widgets.

Folder's structure

  • website-css: LESS & compiled CSS for the site and the CMS overrides
    • fxs_sf_dashboard.less: CMS overrides
    • fxs_screen.less: The main compiler file for the site, we import all LESS files in the strict order:
      • Vendor imports: All external plugins & components styles (LESS if possible).
      • General application styles: LESS vars, LESS mixins, basic HTML styles, and application components.
      • Layouts & pages: Site layouts, Post pages, detail pages.
      • Widgets: All the widgets built for FXS & clients (CTP, Forecast, Calendar, etc).
    • less: Less base styles & components folder
      • vars.less: All custom LESS variables (color, typo, dimensions, etc) and vendor vars overrides (bottom of file).
      • mixins.less: All custom mixins and vendor mixins overrides (bottom of file).
        Heads Up!

        Don't add any output class in this file.

      • reset.less: HTML normalize fixes (use it with caution: add styles to a tag may cause unexpected results).
      • base.less: Basic text and single components styles, the atoms.
      • forms.less: Basic forms styles: form atoms
      • components.less: Complex objects, group of atoms, molecules and organizms.
      • templating.less: Website layout, grid system, columns, etc.
      • featured-entries.less: Post entrie highlights for homes: news, analysis, video, education, ...
      • post.less: Post and final detail pages styles.
      • vendor: Bootstrap LESS components, lesshat, all external plugin styles.
        Heads Up!

        Don't edit directly from vendor files, override styles & vars into the FXS custom LESS files.

  • website-font: Font awesome assets, other icon fonts.
    Heads Up!

    The main font of the website is "Fira Sans" and is imported via Google fonts (head)

  • website-htmldesign: Static HTML templates
  • website-img: Global site images
  • website-js: All js assets for the general site
  • widgets: Assets and plain examples HTML (not layout) for each widget.
    • core: The general assets that works for all widgets.
      • css: core.less => core.min.css
      • js: general js: Bootstrap, jQuery and c3 (charts library).
      • website-font: Font awesome
    • widget: Spescific widget assets
      • css: widget.less => widget.min.css
        Heads Up!

        This LESS will be imported from "fxs_screen.less" in order to have the same CSS. Avoid duplicity!

      • widget-example.html
    • widgetmultiasset: The multiasset version of each widget, the assets are called from the single asset widget.
      • widgetmultiasset-example.html
    • ...
Heads Up!

For the general site, the only LESS file to compile is "fxs_screen.less" => "fxs_screen.min.css". Uncheck the rest of the less files.

For the widgets, the LESS files to compile are "core.less" => "core.min.css" and (each) "widget.less" => "widget.min.css".

Heads Up 2!

If the CMS needs style adjustments, then edit and compile "fxs_sf_dashboard.less" => "fxs_sf_dashboard.min.css"
